# Security

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security reports.

Instead, contact the maintainers by email (see the public marketing site **Contact** section or this repository’s README). Include:

- Description of the issue and impact
- Steps to reproduce (if possible)
- Whether you have a suggested fix

We aim to acknowledge reports within a few business days.

## Scope

This repository powers the **Trust Auditor** A2A service, optional **Stripe payment adapter**, and reference **identity registry** / **transaction log** components. When reporting, note which component and version (commit or image) you tested.

## Safe harbor

We support good-faith security research that follows this disclosure process and does not violate applicable law.
